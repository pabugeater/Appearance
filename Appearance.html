Appearance
 
1.fixme

Simple Light / Dark / JavaScript / HTML / Objective-C Appearance code for iOS, macOS and Safari. Touch the Appearance icon to toggle the window's appearance, or type 'a'.

Topics

Usage
  • Add this handful of additional statements to your HTML:
    1. Head:
      <link rel="stylesheet" type="text/css" href="https://www.bigcatos.com/BigCatOs/styles/com_bigcatos_styles.css" />
      <script type="text/javascript" src="https://www.bigcatos.com/BigCatOs/js/com_bigcatos_Appearance.js"></script>
    2. Body:
      <body class=container>
    3. Image onclick:
      onclick="com_bigcatos_doDarkMode( event )"
    4. Inject help div:
      <div id=com_bigcatos_darkModeHelpDiv style="display:none; "></div>
    5. Initialization:
      • If not using the Appearance Objective-c wrapper, e.g. a straight HTML page, you must explicitly initialize immediately before the </body> statement:
        <script>com_bigcatos_doDarkMode( Type );</script>
      • If using the Appearance class in an iOS or macOS App, initialization is done implicity on your behalf via a JavaScript call similar to this:
        com_bigcatos_setExplicitAppearance( "dark", "DarkModeHelpDisplayed" );
      Type may be one of these integers:
      • -1 macOS
      • -2 iOS
      • -3 BigCatOs home page star
      • -4 Solar Coaster start in Dark mode
      • -5 Solar Coaster start in Light mode

  • Typical Objective-C usage:
        #include "Appearance.h"
    Appearance *ap;
    ap = [[Appearance alloc] initWithFile:@"Appearance.html" contentController:self.view andFrame:self.view.bounds];
    [self.view addSubview:ap];
    [self addAppearanceConstraintsForView:self.view];
  • Build notes:
    • To compile the Appearance class in macOS add kAppearanceX=1 to the target's C Preprocessor section.
    • Force source file Appearance.m into bundle Resources folder via a custom build phase Copy Files script. It's impossible to do this in standard Copy Bundle Resources phase. Required for Appearance.sh.

  • Export note:
    Use File / Export to export Appearance files with substituted version numbers, suitable for distribution. This essentially runs the following command: 
        Appearance.sh srcFolder/AppearanceX.app/Contents/Resources destFolder/Appearance-2018.12.14-08.48.03 '1.3(3)' \
    Appearance.sh Appearance.h Appearance.m Appearance.html com_bigcatos_Appearance.js
